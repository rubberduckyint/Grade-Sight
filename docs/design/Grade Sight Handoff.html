IF NO TOKEN FITS — STOP AND ASK

Every color, spacing, and type decision in this doc is a token. If you're about to hand-pick a hex, a pixel, or a font size that isn't here: don't. Ask instead. That's how the system stays coherent.

Principles

Five rules that settle 90% of decisions.

1 · 18px is the floor.

Body text is 18px (1rem). Never shrink to fit. Density is a layout decision — shorten copy or widen the container, never reduce the font.

2 · Amber is precious.

Only at diagnostic-insight moments. Never as chrome, decoration, or a generic warning color.

3 · Red is evidence only.

Only for mock student-work error marks. Never a UI error state — those use ink + mono microcopy.

4 · Serif for meaning, sans for doing.

Headlines, voice, and quotes are Source Serif 4. Buttons, forms, tables are Inter.

5 · Focus rings never hide.

:focus-visible is enforced globally. Do not remove the outline on any interactive element.

Type

Four Google Fonts. One scale.

Load every font via next/font/google in app/layout.tsx. No font licensing work required.

// app/layout.tsx
import { Source_Serif_4, Inter, JetBrains_Mono, Caveat } from "next/font/google";

const serif = Source_Serif_4({ subsets: ["latin"], variable: "--font-serif", display: "swap" });
const sans  = Inter({          subsets: ["latin"], variable: "--font-sans",  display: "swap" });
const mono  = JetBrains_Mono({ subsets: ["latin"], variable: "--font-mono",  display: "swap" });
const hand  = Caveat({         subsets: ["latin"], variable: "--font-hand",  display: "swap", weight: ["500","600"] });

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <ClerkProvider>
      <html lang="en" className={`${serif.variable} ${sans.variable} ${mono.variable} ${hand.variable} h-full antialiased`}>
        <body className="min-h-full flex flex-col">{children}</body>
      </html>
    </ClerkProvider>
  );
}

Button conventions

DO

Primary = bg-ink text-paper · Secondary = border-rule, paper bg · Tertiary = text link, accent color. Radius is --radius-sm (3px) everywhere.

DON'T

No gradients. No shadows on buttons. No pill buttons. No icon-only primary CTAs — every CTA has a verb.

TrialBanner prop contract

type TrialBannerProps = {
  daysRemaining: number;    // <= 0 hides banner (paywall takes over)
  role: "parent" | "teacher";
  dismissable?: boolean;    // false once <= 1 day
  onAddCard: () => void;
  onDismiss?: () => void;
};

// Tone picks itself:
//   > 7 days         → render nothing
//   4–7 days         → tone="calm"    (paper-soft bg)
//   2–3 days         → tone="insight" (insight-soft bg, amber rule)
//   <= 1 day         → tone="urgent"  (paper-deep bg, mark-colored eyebrow)
Copy patterns

The voice, in examples.

Grade Sight voice

  • Specific > generic. "Your trial ends in 5 days" beats "Your trial is ending soon."
  • Honest about consequences. "You'll lose access tomorrow" not "Features may be limited."
  • Warm but not cute. No emoji. No "Oopsie."
  • Second person. Not "users." You.
  • Never clinical about students. A student's work is evidence, not a diagnosis of them.

Parent vs teacher

  • Parent: text-lg (20px) lead, more white space, "your kid" / "your student."
  • Teacher: text-base (18px), denser data, "your students" / "your class."
  • Same components. Different copy and density. Never build parallel components.

Accessibility

WCAG AA, non-negotiable.

This is a hard requirement, not a nice-to-have. Edtech procurement will check.

  • Base 18px. Gives every text token headroom for AA. Don't shrink below 15px ever.
  • Focus ring mandatory. :focus-visible in globals.css applies site-wide. Do not override outline: none on any interactive element.
  • Contrast check. ink on paper ≈ 13:1. ink-soft on paper ≈ 7:1. accent on paper ≈ 7:1. All pass AA large + normal text.
  • No color-only meaning. Every amber insight also has a mono eyebrow label. Every error state has text, not just red.
  • Keyboard paths. Every CTA reachable by Tab in visual order. Banners are dismissable by Esc when focus is inside.
  • Form labels. Every input has a visible label. Placeholders are not labels.
  • prefers-reduced-motion. Respect it. Skeletons have no shimmer. Transitions cap at 200ms.
  • Semantic HTML. <button> for actions, <a> for navigation. Never <div onClick>.

Before merging any PR

Seven-item checklist.

  • Every font size is a token (--text-*). No raw pixels for type.
  • Every color is a token. If you picked a hex, it's wrong.
  • Visible focus ring on every interactive element (Tab through the page).
  • Amber appears only at a diagnostic-insight moment. Red only on mock student marks.
  • Body text is 18px. Nothing you authored is below 15px.
  • Serif = meaning (headlines, voice). Sans = doing (buttons, forms).
  • Matches the reference canvas. If it doesn't, stop and flag.
